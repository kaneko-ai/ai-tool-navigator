# Task

入力された記事ドラフトについて、事実誤認・誇張・規約リスク・身バレリスク・SEO 損傷を漏れなく指摘してください。

# Check points

## 1. AI ツール・サービスの情報正確性
- AI ツール（GenSpark / Claude / Gemini / GitHub Copilot / ChatGPT / 各種 AI スクール）の料金・仕様・モデル名・リリース日
- スクール名・サービス名のスペル
- 給付金対象・補助金条件の記述

## 2. 数値・統計の出典
- 年収・単価・利用者数・割合 などの数値に出典 URL が併記されているか
- URL が実在する公式・公的統計か（en-japan / PR TIMES / 求人ボックス / doda / レバテック / 各サービス公式 等）
- 出典なき数値があれば「要出典」または「該当数値を削除」と指摘

## 3. 誇大表現・根拠なき断定（NG パターン）
- 「最速」「業界 No.1」「絶対」「100% 稼げる」「必ず合格」「圧倒的」等
- 比較最上級は出典がない限り NG

## 4. 検証不能な経歴設定
- 「私は元 SIer で年収 1200 万円」「元 Google エンジニア」等の検証不能な一人称
- 全て指摘して削除を提案

## 5. 身バレ・特定リスク
- 大学名・研究室名・実在企業の社員名・特定個人名
- 投稿時間帯・地域が特定できる記述

## 6. AI 開示文
- 本文冒頭（リード文直後）に AI 活用の明示があるか
- 「※本記事は AI を活用して下書きを作成し、人手で校閲・加筆しています。」相当の文言があるか

## 7. 末尾の句点切れ
- 本文最後が「。」「！」「？」「）」のいずれかで終わっているか
- メタ情報セクションの前の本文末尾を確認

## 8. SEO 構造
- description フィールドに H2 (`##`) が混入していないか
- title が長すぎないか（32 文字以内推奨）
- H2 が 4-6 個あるか
- H2 の中身が空でないか

## 9. CTA とアフィリエイト規約
- moshimo CTA プレースホルダ `<!-- CTA:MOSHIMO_CONOHA_WING -->` が 1-2 箇所配置されているか
- CTA 周辺に誇大表現がないか
- アフィリエイト関連法令（景品表示法・特定商取引法）に抵触しそうな断定がないか
- A8 提携先への直接リンクが入っていないか（審査未完了のため NG）

## 10. 文字数
- 本文 2500-3500 字の範囲か
- 短すぎ／長すぎを指摘
