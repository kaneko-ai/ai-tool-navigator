## 重大な問題（修正必須）
- [問題1]: 「レバテックの 2024 年調査によると、エンジニア向けフリーランス案件の平均月単価は 79.9 万円で、これを月 20 日稼働で割ると**月 4 万円前後の時給換算**になります（[レバテック フリーランス](https://levtech.jp/)）。」
  → 指摘: 計算と単位が誤っている（「月 4 万円の時給換算」は意味不明）。正しい計算例を併記するか出典ページの該当箇所へ直接リンクを貼る必要あり。修正案: 「レバテックの統計によれば平均月単価は約79.9万円（※出典の該当ページを明記）で、これを月20日・1日8時間で割ると時給換算は約5,000円/時（79.9万円 ÷ 160時間 ≒ 4,994円/時）になります。出典: レバテック統計ページ（該当URLを明記）」
  → 参考（出典必須）: https://freelance.levtech.jp/ または該当統計ページ（編集時に正確URLを貼ること）

- [問題2]: 「初期段階の月 2〜5 万円…6 ヶ月後に 15〜25 万円、1 年後に 30〜50 万円に伸びるケースが実際に報告されています。」
  → 指摘: 出典なしの推移データで「実際に報告されています」と断定している。根拠が無ければ削除または「事例によっては〜のケースがある（出典：xxx）」と必ず出典をつけること。修正案: 「事例報告では…（出典URL）」または数値を削除。

- [問題3]: 「第一は、3 ヶ月連続で月 50 万円以上の副業収入を確保すること」「副業時代の 1.2 倍の稼ぎが必要」
  → 指摘: 「3ヶ月連続」「1.2倍」等の閾値には根拠が示されていない。勧告という形なら「目安」へ表現変更し、算出根拠（税負担増や社会保険負担のモデル計算）を示すこと。修正案: 「目安として3ヶ月連続～。根拠：想定税率・社会保険の増加分を試算した場合（試算テーブルを添付）」

- [問題4]: ヘッダー画像プロンプトに実在作家名を指定: 「Loundraw or Mogami Tokihito style」
  → 指摘: 実在アーティスト名をスタイル指定すると著作権・倫理上のリスクがある。文化庁の著作権に関する注意を参考に、作家名は削除し「ソフトパステル風」「日本の現代イラスト風」等の非特定スタイルに置き換えること。参考: 文化庁 著作権 http://www.bunka.go.jp/seisaku/chosakuken/

- [問題5]: AI ツール・サービス（GenSpark / Claude / Gemini / GitHub Copilot / ChatGPT / 各種 AI スクール）についての料金・仕様・モデル名・リリース日の明記が不足
  → 指摘: 検索対象チェックポイントに挙げられているサービスの具体的な料金やモデル名、リリース日は本文にない。各主張に対して公式ソース（企業公式FAQ/価格ページ・リリースノート）を必ず明示し、古い情報は更新すること（例: Copilot料金は GitHub 公式、GeminiはGoogle公式等）。

## 軽微な問題（修正推奨）
- [問題6]: 「多くの人は再投資に 30%、消費に 20%、貯蓄に 50% を充てている」  
  → 指摘: 出典がない比率。読者向けの「モデル配分」なら「例: 再投資30%等」と明示するか、根拠となる調査URLを追加する。

- [問題7]: NISA の説明はおおむね正確だが「年利 5%」前提での将来値は仮定
  → 指摘: 数値シミュレーションは仮定（年利5%・毎月10万円）。修正案: 使った前提（年利・複利計算方法・投資タイミング）を明記し、FSAのNISA制度ページを明記する。参考: https://www.fsa.go.jp/nisa/

- [問題8]: 「TechClips ME、ミライフ、レバテックといったサービスは…スキル〜単価〜独立タイミングを統合的に管理するシステムとして機能しています」
  → 指摘: サービスごとに機能差があるため一般化しすぎ。各社機能（スカウト、独立支援、診断ツール）の該当ページを引用して「〜のような機能がある場合が多い」と表現を緩める。

- [問題9]: 見出し（title）の長さと meta description の注意
  → 指摘: title が長すぎる可能性（32文字推奨）。meta description に H2 が混入していないか確認する。修正案: タイトル短縮例「AI副業で新卒年収1.5倍へ：独立判断と資産戦略」

- [問題10]: 誇大表現の緩和
  → 指摘: 「決して夢ではなく、〜12〜18ヶ月で実現可能な目標です。」は過度に断定的。修正案: 「ケースによっては12～18ヶ月で実現できる場合があるが、個人差が大きい」と条件付きにする。

- [問題11]: 出典リストの不備
  → 指摘: レバテックの具体ページ、TechClips ME のサービスページ、ミライフの公式ページなど「該当する統計や機能説明ページ」への直接リンクを出典リストに追加すること（単なるトップページURLだけでは不十分）。
